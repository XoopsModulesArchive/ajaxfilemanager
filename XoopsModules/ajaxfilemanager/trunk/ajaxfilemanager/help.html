Install

Xoops patches

This module uses Xoops Preload features.
It replaces the official version of Image Manager.
To run the module is necessary to apply some changes to the official version (>2.4.2) of Xoops.
In "extra/_PATCHED" directory there are patched xoops files.

  • image._PATCHED.php
    overwrite "XOOPS_ROOT_PATH/image.php" Xoops file
  • imagemanager._PATCHED.php
    overwrite "XOOPS_ROOT_PATH/imagemanager.php" Xoops file
  • modules.system.admin.images.main._PATCHED.php
    overwrite "XOOPS_ROOT_PATH/modules/system/admin/images/main.php" Xoops file

Image manager Admin

Popup Image manager

You can choose (module options) 'standard' or 'enhanced' popup image manager.

Textsanitizer [newimg] tag

You can use [newimg] tag in any Xoops dhtmltextarea filed, if properly installed and activated.
Is possible to install this textsanitizer extention from Newimagemanager administrative menu.

Syntax

[newimg options ] image_id [/newimg]

Where:

  • options are image.php options
  • image_id is New Image Manager image id

Exemples

  • [newimg width=100 ] 3 [/newimg] will insert a resized copy of New Image Manager image with id=3
  • [newimg ] 5 [/newimg] will insert New Image Manager image with id=5

image.php

This module uses an enhanced "image.php" script version. This enhanced version is very useful in many cases, for example when you need a smallest version of a uplaoded image. The new "image.php" script uses some "on fly" resize features... with a very interesting cache system.  ;-)

You MUST TRY IT!

Parameters need to be passed in through the URL's query string:

  • image_id OR id New Image Manager Xoops module image id;
  • image path relative to XOOPS_MAIN_PATH, path of local image starting with "/" (e.g. /images/toast.jpg);
  • width maximum width of final image in pixels (e.g. 700);
  • heigh t maximum height of final image in pixels (e.g. 700);
  • color (optional) background hex color for filling transparent PNGs (e.g. 900 or 16a942);
  • cropratio (optional) ratio of width to height to crop final image (e.g. 1:1 or 3:2);
  • nocache (optional) does not read image from the cache;
  • noservercache (optional); // STILL TO DO
  • nobrowsercache (optional); // STILL TO DO
  • quality (optional, 0-100, default: 90) quality of output image;
  • filter (optional, PHP5+ imagefilter 2nd, 3rd, 4th, 5th arguments, more info on php.net manual) a filter or an array of filters;
  • radius (optional, 1, 2, 3 or 4 integer values, CW) round corner radius

Examples

Resizing a JPEG :
<img src="/image.php? image =image-name.jpg? width =100& height =100" alt="Don't forget your alt text" />
Resizing and cropping a JPEG into a square :
<img src="/image.php/image-name.jpg? width =100& height =100& cropratio =1:1" alt="Don't forget your alt text" />
Matting a PNG with #990000 :
<img src="/image.php/image-name.png? color =900& image =/path/to/image.png" alt="Don't forget your alt text" />
Apply a filter :
<img src="/image.php? image =/path/to/image.png& filter =IMG_FILTER_COLORIZE,128,60,256" alt="Don't forget your alt text" />
Apply more filters (array) :
<img src="/image.php? image =/path/to/image.png& filter[] =IMG_FILTER_GRAYSCALE& filter[] =IMG_FILTER_COLORIZE,128,60,256" alt="Don't forget your alt text" />
Round the image corners :
All corners with same radius
<img src="/image.php? image =/path/to/image.png& radius =20" alt="Don't forget your alt text" />
Left and right corners with different radius (20 for left corners and 40 for right corners)
<img src="/image.php? image =/path/to/image.png& radius =20,40" alt="Don't forget your alt text" />
4 corners, 4 radius, clockwise order
<img src="/image.php? image =/path/to/image.png& radius =20,40,0,10" alt="Don't forget your alt text" />
