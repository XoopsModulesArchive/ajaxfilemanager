Textsanitizer [video] tag/extension

You MUST TRY IT!

With this extension you can insert a Flash FLV video into your page.
You can use [video] tag in any Xoops dhtmltextarea filed, if properly installed and activated.

How to install

Is possible to install this textsanitizer extension from Ajaxfilemanager administrative menu / Extensions Manager.

Syntax

[video options]video_url[/video]

Where:

  • options are writen as opt1=val1,opt2=val2, ...
  • video is .flv video absolute url

Exemples

  • [video width=100,height=100]my_video_url[/video] will insert a 100x100 video
  • [video]my_video_url[/video] will insert a default sizes video

Textsanitizer [gmap] tag/extension

You MUST TRY IT!

With this extension you can insert a Google KML map into your page.

You can use [gmap] tag in any Xoops dhtmltextarea filed, if extention is properly installed and activated.
Instead of developing your own page with Google Maps API, it's possible to write a KML file and send it to Google Maps.
Google Maps understands a small subset of KML 2.0, and implements markers, polylines, polygons, info windows and a sidebar. A simple KML file contains an XML header, a KML header and a <Document>...</Document> structure. That <Document> can contain a series of <Placemark>s which each describe one marker or polyline.

Here's what a basic XML file looks like

<?xml version="1.0" encoding="UTF-8"?>
<kml xmlns="http://earth.google.com/kml/2.0">
<Document>
<name>KML Example file</name>
<description>Simple markers</description>

<Placemark>
<name>Marker 1</name>
<description>Some stuff to put in the first info window</description>
<Point>
<coordinates>-122.1,37.4,0</coordinates>
</Point>
</Placemark>

</Document>
</kml>
  • The whole file must be syntactically valid XML. The case of the tags is significant.
  • The Document name is displayed in the title bar and in bold in the sidebar.
  • The Document description is displayed in the sidebar.
  • The Placemark name is displayed in the sidebar and in bold in the info window.
  • The Placemark description is displayed in the info window.
  • The coordinates are in the order longitude, latitude, altitude. (Note that this is the opposite order from GLatLng() in the Google Maps API) The altitude value is not used.
  • The map will be automatically centred and zoomed in such a way as to contain all the placemarks.
  • KML files can't be sent from your local disk with URLs that begin with "file://" they need to be on a real web server.

You can generate KML files using Google Earth.

How to install

Is possible to install this textsanitizer extension from Ajaxfilemanager administrative menu / Extensions Manager.

Syntax

[gmap options]kml_url[/video]

Where:

  • options are writen as opt1=val1,opt2=val2, ...
  • kml_url is Google Maps .kml file absolute url

Exemples

  • [gmap width=100,height=100]my_kml_url[/gmap] will insert a 100x100 gmap
  • [gmap]my_kml_url[/gmap] will insert a default sizes gmap

image.php

This module uses an enhanced "image.php" script version. This enhanced version is very useful in many cases, for example when you need a smallest version of a uplaoded image. The new "image.php" script uses some "on fly" resize features... with a very interesting cache system.  ;-)

You MUST TRY IT!


Parameters need to be passed in through the URL's query string:

  • image_id OR id New Image Manager Xoops module image id;
  • image path relative to XOOPS_MAIN_PATH, path of local image starting with "/" (e.g. /images/toast.jpg);
  • width maximum width of final image in pixels (e.g. 700);
  • height maximum height of final image in pixels (e.g. 700);
  • color (optional) background hex color for filling transparent PNGs (e.g. 900 or 16a942);
  • cropratio (optional) ratio of width to height to crop final image (e.g. 1:1 or 3:2);
  • nocache (optional) does not read image from the cache;
  • noservercache (optional); // STILL TO DO
  • nobrowsercache (optional); // STILL TO DO
  • quality (optional, 0-100, default: 90) quality of output image;
  • filter(optional, PHP5+ imagefilter 2nd, 3rd, 4th, 5th arguments, more info on php.net manual) a filter or an array of filters;
  • radius (optional, 1, 2, 3 or 4 integer values, CW) round corner radius

Examples

Resizing a JPEG:
<img src="/image.php?image=image-name.jpg&width=100&height=100" alt="Don't forget your alt text" />
Resizing and cropping a JPEG into a square:
<img src="/image.php/image-name.jpg?width=100&height=100&cropratio=1:1" alt="Don't forget your alt text" />
Matting a PNG with #990000:
<img src="/image.php/image-name.png?color=900&image=/path/to/image.png" alt="Don't forget your alt text" />
Apply a filter:
<img src="/image.php?image=/path/to/image.png&filter=IMG_FILTER_COLORIZE,128,60,256" alt="Don't forget your alt text" />
Apply more filters (array):
<img src="/image.php? image=/path/to/image.png&filter[]=IMG_FILTER_GRAYSCALE&filter[]=IMG_FILTER_COLORIZE,128,60,256" alt="Don't forget your alt text" />
Round the image corners:
All corners with same radius:
<img src="/image.php?image=/path/to/image.png&radius=20" alt="Don't forget your alt text" />
Left and right corners with different radius (20 for left corners and 40 for right corners)
<img src="/image.php? image=/path/to/image.png&radius=20,40" alt="Don't forget your alt text" />
4 corners, 4 radius, clockwise order
<img src="/image.php?image=/path/to/image.png&radius=20,40,0,10" alt="Don't forget your alt text" />
